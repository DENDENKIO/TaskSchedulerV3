# TaskSchedulerV3 改修仕様書（既存実装を尊重する差分版）

作成日: 2026-04-17  
目的: 既に実装済みの機能は重複実装させず、未実装または不十分な箇所だけをローカルAIに安全に改修させるための詳細仕様書

---

## 1. この仕様書の前提

この仕様書は、**TaskSchedulerV3 の現在のローカルコードを実際に読んだ上で差分実装する**ことを前提とする。

そのため、以下を絶対条件とする。

- 既に実装済みの機能は再実装しない
- 既に十分な品質で存在する画面やロジックは流用する
- 不十分なUIのみを改修する
- 既存DB構造で足りるなら新テーブルや新カラムを増やしすぎない
- 既存の camera / OCR / PhotoMemo / recurring 関連コードがあるなら活用する

---

## 2. 最初に必ずやる「実装済み確認」

ローカルAIはコード出力の前に、以下を確認してレポート化すること。

### 2.1 確認対象
1. MainActivity / navigation 構成
2. AppDatabase の version と entity 構成
3. Task / Tag / PhotoMemo の model 構成
4. TaskDao / TagDao / PhotoMemoDao の既存クエリ
5. schedulelist 画面の表示内容
6. addtask 画面の保存導線
7. tag 管理画面の色入力有無
8. recurring 関連画面と保存方式
9. quick draft 相当機能の有無
10. widget 関連実装の有無
11. camera / FileProvider / OCR 導線の実装状況

### 2.2 判定分類
各項目は以下で分類する。
- 実装済みで十分
- 実装済みだが改修必要
- 未実装
- 要調査

### 2.3 出力ルール
最初の出力は必ず「現状確認レポート」にし、コードはまだ出さない。

---

## 3. 機能別 差分対象の考え方

### 3.1 一覧タグフィルタ

#### 省く条件
- 既に一覧画面上部にタグフィルタがある
- 単一選択または複数選択で十分に絞り込める
- 表示崩れや使いにくさがない

#### 改修する条件
- タグ絞り込みが別画面にあって遅い
- 一覧画面上から即切り替えできない
- 視認性が低い

#### 必要な仕様
- 一覧上部横スクロールチップ
- 先頭に「すべて」
- 単一選択で開始、将来拡張しやすいstate
- ダーク基調の見やすいチップデザイン

---

### 3.2 一覧表示モード

#### 省く条件
- TODAY / ALL / DONE / RECURRING / DRAFT 相当の切替が既に自然にできる

#### 改修する条件
- 切替が存在しない
- RECURRING と DRAFT が一覧で見られない

#### 必要な仕様
- TODAY
- ALL
- DONE
- RECURRING
- DRAFT

---

### 3.3 一覧行UI

#### 省く条件
- 優先度が視覚的にわかる
- 残日数がわかる
- タグが整理表示される
- 完了状態が見やすい

#### 改修する条件
- 情報が見づらい
- 優先度や期限が埋もれている
- 1行の視認性が低い

#### 必要な仕様
- 左優先度ライン
- タイトル1行優先
- 進捗バー
- タグ最大2件
- 残日数バッジ
- 完了時 muted化

---

### 3.4 addtask の保存ボタン

#### 省く条件
- 既に右上保存になっている
- 下部保存ボタンが残っていない
- スクロール量に依存せず保存できる

#### 改修する条件
- 保存が下部のみ
- 長いフォームで保存が遅い

#### 必要な仕様
- TopBar 右上保存
- 左閉じる / 戻る
- 保存中ローディング
- canSave 制御

---

### 3.5 tag 管理の色設定

#### 省く条件
- 既に色入力UIが存在しない
- タグ見た目が統一されている

#### 改修する条件
- 色入力欄がまだある
- タグ色のばらつきが強い

#### 必要な仕様
- 色入力UI削除
- 固定色保存
- DB color は当面残してよい

---

### 3.6 recurring 管理

#### 省く条件
- 設定画面側で繰り返し予定を管理できる
- N日ごと / 曜日複数 / 毎月日付複数 が実装済み
- 一覧で recurring を確認できる

#### 改修する条件
- recurring が弱い
- 登録種類が不足している
- 設定画面導線がない

#### 必要な仕様
- settings 配下に recurring 管理
- 3ルール対応
- recurrencePattern / recurrenceDays を既存構造に合わせて保存

---

### 3.7 QuickDraft 仮登録

#### 省く条件
- 写真からの仮登録が既に存在する
- 仮登録一覧と編集画面がある
- 本登録変換ができる

#### 改修する条件
- 画像取得後に直接Task化してしまう
- 仮登録管理が弱い
- 本登録前の整理ができない

#### 必要な仕様
- QuickDraftTask 相当の保存先
- 仮登録一覧
- 仮登録編集
- 本登録変換
- OCR / PhotoMemo 連携

---

### 3.8 Widget 高速仮登録

#### 省く条件
- 既に widget からタグ選択→カメラ起動→仮登録保存→管理画面遷移ができる

#### 改修する条件
- widget がない
- widget はあるが仮登録導線がない
- タグ選択できない
- カメラ即起動しない

#### 必要な仕様
- Widget上でタグ選択または未選択
- カメラボタン押下で即起動
- 撮影成功後に自動で QuickDraft 保存
- 保存後に仮登録管理画面へ遷移
- widgetごとに選択タグ状態保持

## 4. UI仕様（改修が必要な場合のみ適用）

### 4.1 色
- Background: `#0E0E10`
- Surface: `#16161A`
- Surface2: `#1C1C21`
- Accent: `#7C6AFF`
- Danger: `#F46060`
- Warning: `#F0A030`
- Success: `#3DD68C`
- Text: `#E8E8F0`
- TextMuted: `#7A7A8C`

### 4.2 一覧行
- 左端 3dp 優先度ライン
- タイトル 1行優先
- 下部に進捗バー
- タグ最大2件
- 右側に残日数バッジ

### 4.3 右上保存
- TopBar 右上テキストまたはアイコン+テキスト
- 下部保存ボタンは削除

### 4.4 Widget
- 現在タグ表示
- タグ順送り切替ボタン
- カメラボタン
- 未選択状態対応

---

## 5. データ仕様（不足している場合のみ追加）

### 5.1 QuickDraftTask
既存に仮登録テーブルがなければ追加する。

推奨項目:
- id
- title
- description
- photoPath
- ocrText
- tagId
- createdAt
- updatedAt
- status

### 5.2 recurring 保存
既存Taskの recurrencePattern / recurrenceDays / recurrenceEndDate を使えるなら新設しない。

### 5.3 Tag color
DBの color は今回は削除しない。
UI入力のみ廃止する。

---

## 6. 実装優先順位

1. 現状確認レポート
2. QuickDraft基盤の有無確認と不足分補完
3. 一覧画面のタグフィルタ / 表示モード
4. 一覧行UI改善
5. addtask 右上保存化
6. tag 色入力廃止
7. recurring 管理不足分補完
8. widget 高速仮登録
9. 最後に navigation / MainActivity 接続確認

---

## 7. 受け入れ条件

- 既にある機能は重複実装されていない
- 未実装部分のみ追加されている
- 一覧画面の操作効率が上がっている
- 保存導線が右上化されている
- tag 色設定UIが消えている
- recurring が必要十分に使える
- QuickDraft が本登録前の整理に使える
- widget から最速仮登録できる
- 既存 camera / OCR / PhotoMemo が壊れていない
- DB migration が安全
